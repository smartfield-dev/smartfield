Security Audit — Manual Testing

For auditors, pentesters, and security teams. Test each attack vector yourself.

Instructions: Type something in the fields below (e.g. "test@email.com" and "MyPassword123"). Then open DevTools (F12 → Console) and run each test manually. Or click "Run All Tests" to execute them automatically.

Print this page (Ctrl+P) to generate a PDF audit report with results.

15 Attack Vectors

Each test targets a specific attack surface. A secure implementation should block all 15.

#AttackResultStatus

Test 1 — Read .value

Can JavaScript read the plaintext value?

document.querySelector('smart-field').value
EXPECTED: Encrypted payload (base64 string starting with "eyJ2Ijo..."), NOT plaintext.

Test 2 — Access Shadow DOM

Can JavaScript access the internal DOM structure?

document.querySelector('smart-field').shadowRoot
EXPECTED: null (closed Shadow DOM)

Test 3 — Read innerHTML

Can JavaScript extract content from the element?

document.querySelector('smart-field').innerHTML
EXPECTED: "" (empty string)

Test 4 — Read textContent

document.querySelector('smart-field').textContent
EXPECTED: "" (empty string)

Test 5 — Find input elements

Can JavaScript find the real input inside SmartField?

document.querySelector('input[type="password"]')
EXPECTED: Returns the NORMAL input (comparison field), NOT the SmartField input. SmartField's input is hidden inside closed Shadow DOM.

Test 6 — Read metadata

Can JavaScript determine the field type, name, or length?

const sf = document.querySelector('smart-field');
console.log('type:', sf.type);    // → "encrypted"
console.log('name:', sf.name);    // → random ID (sf_a7f3c2...)
console.log('length:', sf.length); // → -1
EXPECTED: type="encrypted", name=random, length=-1. Attacker cannot identify which field is password vs email.

Test 7 — Inject value

Can JavaScript write a value into the field?

const sf = document.querySelector('smart-field');
sf.value = 'INJECTED';
console.log(sf.value); // still encrypted, not "INJECTED"
EXPECTED: .value setter is a no-op. The injected value is ignored.

Test 8 — Access children

const sf = document.querySelector('smart-field');
console.log('children:', sf.children.length);
console.log('childNodes:', sf.childNodes.length);
EXPECTED: 0 children, 0 childNodes

Test 9 — outerHTML extraction

document.querySelector('smart-field').outerHTML
EXPECTED: Shows the tag with attributes but NO plaintext value anywhere.

Test 10 — Object.keys enumeration

Can JavaScript discover sensitive properties?

Object.keys(document.querySelector('smart-field'))
EXPECTED: Empty array or only non-sensitive keys. No "realValue", "password", etc.

Test 11 — JSON.stringify

JSON.stringify(document.querySelector('smart-field'))
EXPECTED: "{}" or minimal object. WeakMap data is invisible to serialization.

Test 12 — Prototype pollution

Can JavaScript override the .value property descriptor?

try {
  const sf = document.querySelector('smart-field');
  Object.defineProperty(sf, 'value', { get: () => 'STOLEN' });
} catch(e) {
  console.log(e.message);
}
EXPECTED: TypeError "Cannot redefine property: value". The property is non-configurable.

Test 13 — MutationObserver

Can JavaScript observe changes inside the Shadow DOM?

const sf = document.querySelector('smart-field');
const obs = new MutationObserver(m => console.log('DETECTED:', m));
obs.observe(sf, { childList: true, subtree: true, characterData: true, attributes: true });
// Type something... observer sees nothing inside closed Shadow DOM
EXPECTED: No mutations detected. Closed Shadow DOM is invisible to external observers.

Test 14 — Keyboard event interception

Can JavaScript capture keystrokes from SmartField?

document.addEventListener('keydown', e => console.log('KEYDOWN:', e.key));
document.addEventListener('keypress', e => console.log('KEYPRESS:', e.key));
document.addEventListener('keyup', e => console.log('KEYUP:', e.key));
// Type in SmartField — events don't propagate
EXPECTED: No keyboard events captured. SmartField uses stopPropagation() on all keyboard events.

Test 15 — Clipboard extraction

Can JavaScript copy the field content?

// Try selecting and copying
document.execCommand('selectAll');
document.execCommand('copy');
// Try reading clipboard
navigator.clipboard.readText().then(t => console.log('Clipboard:', t));
EXPECTED: Clipboard contains nothing from SmartField. Copy, cut, paste, select are all blocked.

Comparison: SmartField vs Normal Input

Run this to see the difference side by side:

const sf = document.querySelector('smart-field');
const ni = document.getElementById('normal-input');

console.log('--- SMARTFIELD ---');
console.log('.value:', sf.value);           // encrypted
console.log('.shadowRoot:', sf.shadowRoot); // null
console.log('.type:', sf.type);             // "encrypted"

console.log('--- NORMAL INPUT ---');
console.log('.value:', ni.value);           // PLAINTEXT!
console.log('.type:', ni.type);             // "password"
console.log('Visible to Hotjar: YES');
