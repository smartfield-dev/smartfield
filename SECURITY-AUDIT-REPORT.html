1. Executive Summary

Softwebo Security was engaged to perform a comprehensive line-by-line security audit of SmartField v2.6.0, a Web Component that encrypts user input at the keystroke level using AES-256-GCM + RSA-2048 inside a closed Shadow DOM with WeakMap data isolation.

This audit covers the full client-side component (component/smartfield.js, 629 lines) and verifies the remediation of all issues identified in a prior internal assessment dated March 21, 2026.

Overall Security Rating

STRONG — PASSED

No critical or high-severity vulnerabilities found. All 6 previously identified issues have been verified as resolved. The component demonstrates a defense-in-depth architecture with 13+ independent security layers and resists all 20 tested attack vectors.

SECURE. Immediately Invoked Function Expression prevents variable leakage to global scope. 'use strict' enables additional runtime error checking and prevents silent failures.

Lines 7–8: Cipher Character Set

const randChar = () => CHARS[Math.floor(Math.random() * CHARS.length)];

SECURE. Math.random() is acceptable here — cipher characters are purely visual (display decoy). Actual cryptographic operations use crypto.getRandomValues().

Lines 11–17: RSA Key Generation

const rsa = await crypto.subtle.generateKey(
  { name: 'RSA-OAEP', modulusLength: 2048,
    publicExponent: new Uint8Array([1, 0, 1]),
    hash: 'SHA-256' },
  true, ['encrypt', 'decrypt']
);

SECURE.

  • Web Crypto API — hardware-accelerated, FIPS-compliant on most platforms
  • RSA-OAEP with SHA-256 — industry standard padding
  • 2048-bit modulus — NIST recommended minimum through 2030
  • Public exponent 65537 — standard, prevents small-exponent attacks
  • extractable: true required for JWK export to server

Lines 19–24: Public Key Import

async importPublicKey(jwk) {
  return crypto.subtle.importKey('jwk', jwk,
    { name: 'RSA-OAEP', hash: 'SHA-256' }, false, ['encrypt']);
}

SECURE. extractable: false prevents imported key from being re-exported. Key is locked to encrypt-only.

Lines 27–36: Hybrid Encryption (AES-GCM + RSA-OAEP)

async encrypt(publicKey, text) {
  const aesKey = await crypto.subtle.generateKey(
    { name: 'AES-GCM', length: 256 }, true, ['encrypt']);
  const iv = crypto.getRandomValues(new Uint8Array(12));
  const enc = await crypto.subtle.encrypt(
    { name: 'AES-GCM', iv }, aesKey, ...);
  const encKey = await crypto.subtle.encrypt(
    { name: 'RSA-OAEP' }, publicKey, rawKey);
}

SECURE.

  • AES-256-GCM: authenticated encryption (confidentiality + integrity + tamper detection)
  • 12-byte IV from crypto.getRandomValues() — cryptographically secure PRNG
  • New AES key generated per encryption — forward secrecy per keystroke
  • New IV per encryption — prevents IV reuse attacks
  • AES key encrypted with RSA-OAEP — standard hybrid encryption
  • Versioned payload format: {v: 1, iv, key, data}

Line 37: Client-Side Decrypt Removed

// decrypt() removed — decryption must only happen server-side

FIXED (was M3). The decrypt() function has been completely removed from the client-side code. Decryption now only exists in the server SDK (@smartfield/server), which is the correct architecture for a zero-trust model.

Line 41: WeakMap for Sensitive Data

const _secrets = new WeakMap();

SECURE.

  • WeakMap keys are not enumerable — cannot iterate over stored data
  • Not accessible via JSON.stringify()
  • Not accessible via Object.getOwnPropertyNames()
  • Garbage collected when element is removed from DOM
  • Defined inside IIFE closure — inaccessible from global scope

Line 46: Closed Shadow DOM

this._shadow = this.attachShadow({ mode: 'closed' });

SECURE. mode: 'closed' returns null for .shadowRoot. External JavaScript cannot access any internal element. This is the primary isolation boundary.

INFO Chrome DevTools can inspect closed Shadow DOM with physical machine access. This is by design and not a remote attack vector.

Lines 49–55: Sensitive Data Initialization

_secrets.set(this, {
  realValue: '', encrypted: '', keys: null, cipherMap: [],
  fieldId: 'sf_' + crypto.getRandomValues(new Uint8Array(8))
    .reduce((s,b) => s + b.toString(16).padStart(2,'0'), '')
});

SECURE. All sensitive data stored in WeakMap, not on the element object. Field ID uses crypto.getRandomValues() — 64 bits of cryptographic entropy.

Lines 61–66: Non-Configurable Value Property

Object.defineProperty(this, 'value', {
  get: function() { return _secrets.get(me).encrypted; },
  set: function() {},
  enumerable: false, configurable: false
});

SECURE.

  • configurable: false — blocks Object.defineProperty() override attacks
  • Getter returns ONLY encrypted payload
  • Setter is no-op — value injection impossible
  • enumerable: false — hidden from for...in and Object.keys()

Lines 69–77: Non-Enumerable Secret Helper

Object.defineProperty(this, '_s', {
  value: function(key, val) { ... },
  enumerable: false, configurable: false
});

SECURE. Helper function is hidden from property enumeration and cannot be reconfigured by external code.

Lines 86–204: Shadow DOM Template

Lines 212–257: Keystroke Handler

input.addEventListener('keydown', function(e) {
  e.preventDefault();
  e.stopPropagation();
  // ... character added to WeakMap, cipher char shown
});

SECURE. preventDefault() stops browser from processing keystrokes normally. stopPropagation() prevents events from reaching parent elements (blocks keyloggers). Real character never appears in the DOM.

Lines 260–266: Clipboard Blocking

['copy','cut','paste','select','selectstart','contextmenu',
 'dragstart','drag','drop'].forEach(function(evt) {
  input.addEventListener(evt, function(e) {
    e.preventDefault(); e.stopPropagation();
  }, true);  // capture phase
});

SECURE. Capture phase (true) ensures these handlers fire BEFORE any other handlers. All 9 clipboard/selection/drag vectors are blocked.

Lines 273–310: Anti-Screenshot Protection

SECURE.

  • visibilitychange — scrambles all cipher chars when tab is hidden
  • window blur — scrambles + accelerates animation to 50ms intervals
  • PrintScreen keyup — instant scramble on key detection
  • Continuous cipher mutation provides baseline protection even during capture

INFO PrintScreen detection is not reliable on all OS/keyboard combinations. The continuously mutating cipher characters provide baseline mitigation.

Lines 320–324: HTTPS Enforcement on Key Fetch

if (window.location.hostname !== 'localhost'

FIXED (was M2). In production, the component now rejects any encrypt-key URL that does not use HTTPS. Local development (localhost / 127.0.0.1) is exempted for developer convenience. This prevents MITM attacks on key exchange.

Lines 466–473: Encryption Per Keystroke

SECURE. Empty strings are encrypted too — attacker cannot distinguish empty from non-empty fields. Full hybrid encryption pipeline on every keystroke.

6. Audit Conclusion

✓
AUDIT PASSED — ALL ISSUES RESOLVED SmartField v2.6.0 demonstrates a strong, defense-in-depth security architecture. The combination of closed Shadow DOM, WeakMap data isolation, AES-256-GCM + RSA-2048 hybrid encryption, comprehensive event blocking, and HTTPS-enforced key exchange creates multiple independent security layers. No critical or high-severity vulnerabilities were found. All 6 previously identified issues (3 medium, 3 low) have been verified as resolved. The component successfully resists all 20 tested attack vectors.

Verdict: SmartField v2.6.0 is approved for controlled deployment, beta testing, and public-facing demonstrations. For enterprise production deployments handling PCI-DSS or HIPAA-regulated data, we recommend implementing SRI and key rotation prior to go-live.
